## 2. 面试问题分类体系

通过对 1,686 条面试官发言的分析，面试问题可分为以下六大类：

### 2.1 分类框架

```
面试问题
├── 技术类 (~80%)
│   ├── 八股文/基础知识 — 考察理论功底
│   ├── 项目深挖 — 追问实习/项目细节
│   ├── 系统设计/场景题 — 考察工程落地能力
│   └── 技术追问（大量即时追问）
├── 行为类 (~7%)
│   ├── 行为面试(BQ) — 团队合作、职业规划
│   └── 开放/综合问题 — 自我介绍、反问环节
└── HR类 (~4%)
    └── HR问题 — offer情况、薪资期望、工作偏好
```

### 2.2 各公司问题类型偏好

**技术深度型（项目深挖+八股文 占比高）：**
- **百度**：项目深挖 25 + 八股文 21 = 46 条，技术密度极高
- **腾讯**：项目深挖 27 + 八股文 14 = 41 条
- **饿了么**：项目深挖 8 + 八股文 20 = 28 条，八股文比重大
- **快手**(3场合计)：项目深挖 23 + 八股文 20 = 43 条
- **众安保险+众安**：项目深挖 35 + 八股文 32 = 67 条（4场面试深度极高）
- **字节**：项目深挖 13 + 八股文 9 = 22 条（仅1场一面）

**HR/文化导向型：**
- **芯片厂HR面**：HR问题 8 + 行为面试 6 = 14 条（面试官大量介绍公司）
- **深度智信（MetaGPT）HR面**：HR问题 10 + 行为面试 3 = 13 条
- **创维酷开群面**：开放/综合问题 16 + 行为面试 7 = 23 条

#### 3.1.1 大模型训练与微调（最高频）

| 知识点 | 出现频次 | 典型问题 |
|--------|----------|----------|
| **Attention机制** | ★★☆☆☆ | 多在项目追问中涉及 |

### 3.2 八股文考察趋势总结

1. **大模型时代的"新八股"已取代传统八股**：SFT、RLHF、知识蒸馏、GRPO/DAPO、模型量化、vLLM 成为高频考点，传统的 CNN/RNN/SVM 等几乎不出现
2. **八股文与项目深度绑定**：大多数八股文不是孤立提问，而是在项目追问中自然引出（如"你们SFT的时候用了什么loss？"）
3. **量化知识考察很细**：字节一面中面试官追问到"对称 vs 非对称量化""per-token vs per-channel"的粒度
4. **RAG工程细节是高频考点**：多路召回、chunk策略、query改写、rerank 等工程实践被反复追问
5. **编程题出现但不多**：约出现在拼多多（LRU缓存）、百度（动态规划）等少数面试中

### 4.2 滴滴项目追问热点（最高频项目）

滴滴判责项目是几乎每场面试必问的核心项目。面试官的追问角度可归纳为以下维度：

#### A. 业务理解层

- "这个不应该由主动发起方决定吗？为什么要用模型？"（百度一面）

- "为什么是大模型做而不是规则？"（得物一面）

#### B. 技术方案层

- "技术方案是怎么做的？做到什么效果？"（饿了么二面）

- "是直接拿R1数据做SFT吗？"（字节一面）
- "SFT之后又做了强化学习，采用什么算法？为什么选DAPO？"（拼多多一面）
- "你们用的千问3-8B，一次请求时间大概多少毫秒？"（创维酷开一面）

#### C. 知识蒸馏细节

- "知识蒸馏具体是怎么设计的？"（字节一面）
- "R1生成的数据是否正确，你们怎么判断？"（字节一面）
- "为什么用更小的千问3-8B？"（拼多多二面）
- "有对比32B和8B的效果差异吗？"（Nodesk）

#### D. 强化学习细节

- "DAPO相比GRPO做了哪些变化？"（滴滴二面）
- "GRPO效果不好体现在哪里？"（滴滴二面）
- "reward是怎么设计的？"（滴滴二面）

#### E. 效果与指标

- "80.2%到86.5%是什么模型对比什么策略？"（百度一面）

- "线上QPS有多高？日处理量多少？"（多场提及100万条/日）

### 4.5 面试官项目追问模式总结

面试官的追问遵循一个相对固定的模式：

```
1. 项目概述 → "你先讲讲这个项目/实习做了什么"
2. 业务理解 → "为什么要做这件事？""业务背景是什么？"
3. 技术方案 → "具体怎么做的？""用了什么模型/方法？"
4. 数据细节 → "数据怎么来的？""标注怎么做的？""数据量多少？"
5. 效果指标 → "效果怎么样？""提升了多少？""对比baseline如何？"
6. 深挖细节 → "为什么选这个方法？""有试过其他方案吗？""这里有什么问题？"
7. 工程部署 → "线上怎么部署的？""QPS多少？""资源消耗如何？"
```

这个模式在字节、腾讯、百度、快手、滴滴等大厂面试中尤为明显。

## 5. 系统设计与场景题专题

系统设计类问题出现 23 次，占比较低（1.4%），但质量很高。主要出现在以下场景：

### 5.1 典型系统设计题

| 公司 | 问题摘要 |
|------|----------|
| **MetaGPT CEO面** | "如果让你设计智能体的reward system（类似人类多巴胺/内啡肽系统），你会怎么设计？" |
| **MetaGPT CEO面** | "AI的memory system有六个components，你怎么演化出来？" |
| **云圣智能二面** | "假如你是无人机场景的算法总设计师，要求实时性+低成本显卡+识别准确率，怎么设计？" |
| **云圣智能一面** | "语音识别→文本→指令→无人机/车机交互链路中，你的技术如何应用？" |
| **得物一面** | "你怎么保证模型识别出的未知风险确实是期望的未知风险？" |
| **腾讯一面** | "如果这是线上生产的RAG系统，你会怎么修改你们的方案？" |
| **创维酷开群面** | "单用户月成本8元，1000万用户怎么降到1%成本？" |

### 5.2 场景题特点

1. **与候选人经历强相关**：面试官通常基于候选人已讲的项目经验，延伸出设计类问题
2. **考察工程思维**：关注成本、延迟、可扩展性等实际落地因素
3. **高管面偏向开放性设计**：MetaGPT CEO面的题目非常开放和前沿（reward system设计、memory system架构）
4. **二面/三面出现更多**：一面以项目深挖为主，二面三面更多涉及系统设计

### 6.3 HR面试典型模式

HR面试的流程相对固定：

```
1. HR自我介绍 + 岗位/团队情况介绍
2. 候选人补充自我介绍（偏个人兴趣/非技术方面）
3. 实习体验询问
4. offer情况摸底
5. 公司选择标准 / 优先级排序
6. 职业规划
7. 薪资/入职时间沟通
8. 公司情况补充介绍（福利/培训/晋升）
9. 候选人提问
```

### 6.4 各公司HR面风格差异

- **MetaGPT HR面**：非常详细介绍公司融资/产品/福利（月base 120%、餐补600-700/月、房补2000/月），直接谈offer意向
- **芯片厂HR面**：大量介绍公司技术背景/创始人/融资情况/行业前景，偏"卖公司"风格
- **众安HR面**：简短高效，主要摸底offer情况和入职意向
- **万兴科技HR面**：较常规，职业规划+团队匹配度评估
- **深度智信HR面**：偏行为面试风格，关注工作偏好和团队期望
- **创维酷开群面**：CEO亲自参加，介绍AI原生公司战略，风格独特

## 8. 面试者表现分析

### 8.1 核心优势

1. **项目经历丰富且有深度**：滴滴判责系统从R1蒸馏→SFT→GRPO/DAPO→量化部署形成完整链路，面试官反复追问仍能深入回答
2. **技术栈覆盖面广**：涵盖大模型微调(SFT/DPO/RLHF)、知识蒸馏、RAG、多模态、意图识别、模型量化、推理加速等热门方向
3. **有实际上线经验**：日处理100万条QPS的线上系统经验，是应届生中的稀缺亮点
4. **回答有条理**：在自我介绍和项目描述中，通常按"背景→方法→结果"的结构阐述

### 8.2 可改进之处

#### A. 八股文/理论深度

- **量化知识偏浅**：字节一面中，面试官追问"对称vs非对称""per-token vs per-channel"时，候选人回答"这个我没有详细的了解"
- **AWQ理解不准确**：被面试官纠正"AWQ不是这样"
- **PPO理论推导**：百度一面追问PPO为什么加KL散度惩罚、优势函数的作用时，回答偏表面
- **向量度量理解**：滴滴二面追问inner product和欧式距离是否可能反相关时，回答不够清晰

#### B. 项目表达

- **叙述偶尔冗长**：部分面试中自我介绍和项目描述较长，面试官多次需要打断（"不好意思我稍微打断一下""我们一点点来"）
- **数据指标不够精确**：多场面试中，面试官需反复追问具体的precision/recall数值
- **因果关系表述不清**：有时候候选人说了结论但没有解释原因，面试官需要追问"为什么"

#### C. 系统设计能力

- 系统设计题出现次数不多，但在出现时回答深度有限
- 缺少"成本-效果-延迟"的工程权衡讨论
- 对端侧部署（车规级芯片）等硬件相关知识了解较少

### 8.3 面试官反馈信号

从面试官的即时反馈中可以观察到一些积极和需改进的信号：

**积极信号：**
- "觉得这思路挺好"（众安一面）
- "这个确实还是很有价值的一段经历"（MetaGPT HR面）
- 面试官主动类比自家业务场景（饿了么面试末尾提到骑手判责与候选人项目的相似性）
- 多家公司进入HR面/加面阶段，说明技术面通过率高

**需注意信号：**
- "AWQ不是这样"（字节一面，量化知识被纠正）
- "你没有回答我的问题"（腾讯一面，关于为什么用大模型做rerank）
- "你没有注意到他们是这个pattern吗？"（MetaGPT CEO面，暗示候选人论文阅读不够深入）
- 多次被打断说明表达需更简洁

## 9. 关键发现与建议

### 9.1 面试行为关键发现

1. **项目深挖是面试核心**：约 80% 的面试时间花在项目经历的追问上，准备项目细节远比刷八股文重要
2. **"新八股"取代"旧八股"**：SFT/RLHF/GRPO/量化/RAG 已成为 2026 年大模型岗位的标准八股，传统 ML 八股几乎不考
3. **面试官追问深度惊人**：从业务理解→技术方案→数据细节→效果指标→工程部署，每个项目被追问 5-10 轮是常态
4. **知识蒸馏是面试最高频技术话题**：R1→小模型的蒸馏链路几乎每场必问
5. **HR面的核心目的是摸底offer**：90% 的HR面试时间在了解候选人手里有哪些offer、怎么选择
6. **群面形式罕见**：仅创维酷开一场群面，其余均为一对一

### 9.2 对候选人的建议

#### 短期可提升项

- [ ] **量化知识系统复习**：AWQ / GPTQ / SmoothQuant 的区别，对称vs非对称量化，per-token/per-channel/per-tensor 粒度
- [ ] **PPO/GRPO/DAPO 理论推导**：不仅知道怎么用，还要能推导为什么这么设计（KL散度惩罚、clip、优势函数）
- [ ] **向量检索理论**：inner product vs cosine similarity vs L2 distance 的数学关系和适用场景
- [ ] **项目指标精确化**：每个项目的 baseline、各方法的 precision/recall/F1 精确到小数点，形成对比表
- [ ] **表达简洁训练**：自我介绍控制在 2 分钟内，项目描述先给结论再展开

#### 中期可加强项

- [ ] **系统设计能力**：练习"如何设计一个RAG系统""如何设计大模型推理服务""如何设计端侧部署方案"
- [ ] **论文阅读深度**：不只读方法，要理解设计动机和pattern（如MetaGPT CEO面提到的过程奖励pattern）
- [ ] **编程题准备**：虽然出现不多，但拼多多、百度等公司仍有编程题，LeetCode中等难度需保持手感

### 9.3 面试市场观察

1. **候选人竞争力强**：52场面试覆盖几乎所有头部公司，且多家进入多轮面试，说明简历和基础素质过硬
2. **滴滴转正意向已拿到**：多场HR面中提及已有滴滴意向
3. **大模型方向就业市场火热**：几乎所有公司（包括汽车/芯片/保险/教育）都在招大模型算法工程师
4. **创业公司给予更多的发展空间**：MetaGPT、弋途/戈途、云圣智能等公司的面试更开放，对候选人潜力的考察更多
5. **面试轮次多不代表难度高**：众安4轮、创维酷开3轮（含群面），但整体面试深度不如字节/腾讯/百度的单场面试
